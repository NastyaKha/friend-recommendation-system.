Актуальность проекта

Социальная сеть Вконтакте очень популярна и в ней зарегистрировано большое количество людей. Это большая площадка для общения, но самая главная проблема - это подобрать контингент для общения. Хотя платформа уже имеет рекомендательную систему, но она не совершенна.

Проблема существующих систем

Основная настройка существующих систем - искать друзей по общим знакомым или включать фильтрацию по общим признакам. Но это не всегда помогает для нахождения друзей или знакомых.

Цель работы

Выявить недостатки рекомендательной системы друзей в социальной сети Вконтакте и разработать более оптимальный алгоритм для нахождения знакомых и новых знакомств.

Задачи проекта

  1. Рассмотреть принцип работы существующей системы
  2. Выявить ситуации, которые не рассматривает система
  3. Составить алгоритм фильтрации и рассмотреть наиболее оптимальный тип рекомендательной системы для отбора

Анализ предметной области

Система рекомендаций друзей - это автоматизированная система, которая предлагает потенциальных друзей пользователям социальных сетей.

Где используются такие системы?

Система рекомендаций - это система, которая используется для рекомендации ресурсов, которые могут быть интересны пользователю, исходя из интересов и/или предпочтений пользователей.

Предлагаемый алгоритм

Для улучшения рекомендаций предлагается использовать гибридный подход, сочетающий анализ контента и коллаборативную фильтрацию.

Шаг 1: Сбор данных о пользователе

Анализируются интересы, группы, музыкальные предпочтения, места посещения и другие активность пользователя.

Шаг 2: Векторизация признаков

Создание векторных представлений пользователей на основе их интересов и активности.

// Псевдокод для векторизации пользователя
function vectorizeUser(user) {
  vector = [];
  vector.push(user.musicPreferences);   // Музыкальные предпочтения
  vector.push(user.groupAffiliations);  // Участие в группах
  vector.push(user.interests);          // Список интересов
  vector.push(user.geoData);            // Географические данные
  return normalize(vector);             // Нормализация вектора
}

Шаг 3: Поиск похожих пользователей

Использование косинусного сходства для нахождения пользователей с близкими векторами признаков.

// Псевдокод расчета косинусного сходства
function cosineSimilarity(vecA, vecB) {
  dotProduct = 0;
  magnitudeA = 0;
  magnitudeB = 0;
  
  for (i = 0; i < vecA.length; i++) {
    dotProduct += vecA[i] * vecB[i];
    magnitudeA += vecA[i] * vecA[i];
    magnitudeB += vecB[i] * vecB[i];
  }
  
  magnitudeA = sqrt(magnitudeA);
  magnitudeB = sqrt(magnitudeB);
  
  return dotProduct / (magnitudeA * magnitudeB);
}
